int pg_graph_explore ( struct pg_graph *  graph,

struct pg_error **  error

Explore graph starting from a brick.

All bricks linked to the chosen brick will be added to the graph. Note that bricks should be used by only one graph and all bricks inside a graph should have a different name. You can call this function on an altered graph to re-explore it. If you do so, bricks which are not part of the graph anymore will be removed and new linked bricks will be added to graph.

Parameters
graphgraph where explored bricks goes
brick_namebrick name from where to start adding bricks. Set to NULL and graph will any brick to start exploring.
erroris set in case of an error
Returns
0 on success, -1 on error

◆ pg_graph_new()

PG_WARN_UNUSED struct pg_graph* pg_graph_new ( const char *  name,
struct pg_brick *  explore,
struct pg_error **  error 
)

Allocate and init an empty graph.

A graph is like a brick container: you can add bricks to it (push), you can remove bricks (pop) or directly destroy bricks inside graph. Graph are useful to manipulate bricks by their name and not having to store all bricks by ourself.

An important thing to know is that graph (and bricks) are not thread safe:

  • You must not call pg_graph_poll on the same graph in parallel.
  • You must not add the same brick in two different graph and use them in parallel.
Parameters
namegraph's name
exploreif set, explore graph from this brick. See pg_graph_explore for more information. Create a empty graph if set to NULL.
erroris set in case of an error.
Returns
new allocated and initialized graph, NULL on fail.

const char *  west_queue_name,

Split a graph in two separated graph. Communication between graph is done using two queue brick.

Before: [A]-—[B]-—[C]-—[D]

=> split (B, C)

After: [A]-—[B]-—[Queue1] ~ [Queue2]-—[C]-—[D]

Parameters
graphgraph to split. This graph will contain the west part of the graph after split.
east_graph_namename of the newly created graph containing the east part of the graph after split.
west_brick_namename of the brick where to split, this brick will be part of west_graph.
east_brick_namename of the brick where to split, this brick will be part of east_graph.
west_queue_namename of the west queue (with default queue size)
east_queue_namename of the east queue (with default queue size)
erroris set in case of an error
Returns
pointer to newly created graph corresponding to the east part of the graph. NULL on error.
