1 # Packetgraph ?
2 
3 Packetgraph is a library aiming to give the user a tool to build networks graph easily, It's built upon the fast [DPDK library](http://dpdk.org/).
4 
5 \htmlonly <a href="https://travis-ci.org/outscale/packetgraph"> <img src="https://travis-ci.org/outscale/packetgraph.svg?branch=master"> </a>\endhtmlonly
6 
7 The goal of this library is to provide a really EASY interface to
8 build you own DPDK based application using [Network Function Virtualization](https://en.wikipedia.org/wiki/Network_function_virtualization)
9 Everyone is free to use this library to build up their own network application.
10 
11 Once you have created and connected all bricks in you network graph,
12 some bricks will be able to poll a burst of packets (max 64 packets)
13 and let the burst propagate in you graph.
14 
15 Connections between bricks don't store any packets and each burst will
16 propagate in the graph without any copy.
17 
18 Each graph run on one core but you can connect different graph using
19 Queue bricks (which are thread safe). For example, a graph can be
20 split on demand to be run on different core or even merged.
21 
22 If you want a graphical representation of a graph, you can generate a [dot](https://en.wikipedia.org/wiki/DOT_%28graph_description_language%29) output.
23

26 # Available bricks (ipv4/ipv6):
27 
28 - switch: a layer 2 switch
29 - rxtx: setup your own callbacks to get and sent packets
30 - tap: classic kernel virtual interface
31 - vhost: allow to connect a vhost NIC to a virtual machine (virtio based)
32 - firewall: allow traffic filtering passing through it (based on [NPF](https://github.com/rmind/npf))
33 - diode: only let packets pass in one direction
34 - hub: act as a hub device, passing packets to all connected bricks
35 - nic: allow passing packets to a NIC of the system (accelerated by DPDK)
36 - print: a basic print brick to show packets flowing through it
37 - antispoof: a basic mac checking, arp anti-spoofing and ipv6 neighbor discovery anti-spoofing
38 - vtep: VXLAN Virtual Terminal End Point switching packets on virtual LANs, can encapsulate packets over ipv4 or ipv6
39 - queue: temporally store packets between graph
40 - pmtud(ipv4 only): Path MTU Discovery is an implementation of [RFC 1191](https://tools.ietf.org/html/rfc1191)
41 - fragment-ip(ipv4 only): fragment and reassemble packets, currently broken(https://github.com/outscale/packetgraph/issues/401)
42 - user-dipole: setup your own callback in a dipole brick, to filter or implement your own protocol
43 
44 A lot of other bricks can be created, check our [wall](https://github.com/outscale/packetgraph/issues?q=is%3Aopen+is%3Aissue+label%3Awall) ;)

52 # Examples
53 
54 To build and run examples, you may first check how to build Packetgraph below and adjust your configure command before make:
55 ```
56 $ ./configure --with-examples
57 $ make
58 ```
59 
60 To run a specific example, check run scripts in tests directories:
61 ```
62 $ ./examples/switch/run_vhost.sh
63 $ ./examples/switch/run.sh
64 $ ./examples/firewall/run.sh
65 $ ./examples/rxtx/run.sh
66 $ ./examples/dperf/run.sh
67 ...
68 ```

70 # Building
71 
72 You will need to build DPDK before building Packetgraph.
73 
74 ## Install needed tools
75 
76 You may adapt this depending on your Linux distribution:
77 ```
78 $ sudo apt-get install automake libtool libpcap-dev libglib2.0-dev libjemalloc-dev libnuma-dev
79 ```
80 
81 ## Build DPDK
82 
83 ```
84 $ git clone http://dpdk.org/git/dpdk
85 $ cd dpdk
86 $ git checkout -b v18.05 v18.05
87 $ make config T=x86_64-native-linuxapp-gcc
88 ```
89 
90 Note: use `T=x86_64-native-linuxapp-clang` to build with clang
91 
92 Edit build/.config and be sure to set the following parameters to 'y':
93 - CONFIG_RTE_LIBRTE_PMD_PCAP
94 
95 If you don't want to use some special PMD in DPDK requiring kernel headers,
96 you will have to set the following parameters to 'n':
97 - CONFIG_RTE_EAL_IGB_UIO
98 - CONFIG_RTE_KNI_KMOD
99 
100 Once your .config file is read, you can now build dpdk as follows:
101 
102 ```
103 $ make EXTRA_CFLAGS='-fPIC'
104 ```
105 
106 Finally, set `RTE_SDK` environment variable:
107 ```
108 $ export RTE_SDK=$(pwd)
109 ```
110 
111 ## Build packetgraph
112 ```
113 $ git clone https://github.com/outscale/packetgraph.git
114 $ cd packetgraph
115 $ git submodule update --init
116 $ ./autogen.sh
117 $ ./configure
118 $ make
119 $ make install
120 ```
121 
122 Note: to build with clang, you can use `./configure_clang` wrapper instead of `./configure`.
123 
124 Note 2: You need a compiler that support C11 (gcc 4.9 or superior, or clang 3.6 or superior).
